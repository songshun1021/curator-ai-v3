## 一、背景与目标
Curator AI v3 面向中国大陆在校大学生/研究生，目标是打通“简历 -> 岗位投递 -> 面试准备 -> 面试复盘”的求职闭环，并通过本地优先（IndexedDB）和可编辑系统提示词（`/.system/`）实现可持续迭代。

本阶段核心目标：
- 提升新手可用性（减少隐式入口、提高可见反馈）。
- 提升生成链路稳定性（流式、可取消、草稿隔离、输出可定位）。
- 保持架构约束不变（`/api/chat`、Dexie schema、`ResumeData` 不改）。

## 四、典型问题与修复
1. `Maximum update depth exceeded`
- 原因：Zustand v5 下对象 selector 引用不稳定导致重复更新。
- 修复：高频组件改为逐字段 selector 订阅。

2. PDF 导入崩溃 `Object.defineProperty called on non-object`
- 原因：`pdfjs-dist` 顶层静态导入与运行环境副作用冲突。
- 修复：动态懒加载 + 浏览器保护 + 友好错误映射 + 依赖版本锁定。

3. 生成结果“可保存但看不见”
- 原因：仅切状态未做树定位与可视反馈。
- 修复：`openFilePath` 扩展为“展开父链 + 滚动定位 + 高亮提示”。

4. 复盘“已输入仍提示先填写面试原文”
- 原因：判空逻辑未剥离模板文本。
- 修复：引入模板剥离后再判空，避免误报。

5. 定制简历 JSON 渲染失败
- 原因：模型输出混入解释/代码块，非合法 JSON。
- 修复：Prompt 强约束 + 保存前严格校验，失败不落正式文件。

## 五、当前遗留问题与风险
### 1) 功能差距（你本轮需求相关）
- 右上角“新手使用指引（5 步详细流程）”尚未在右上面板落地为固定引导卡。当前仅有：
  - 首页首次弹窗 3 步引导；
  - 右上“下一步行动”列表。
- PDF 双轨资产尚未完整落地：
  - 当前导入后仅写入主简历编辑态；
  - 尚未持久化 `/简历/个人简历.pdf` 与 `/简历/个人简历.提取.md`；
  - `buildContext` 尚未优先注入“导入简历提取文本”。

### 2) 工程风险
- 仓库存在中文编码污染残留（个别源码/文档出现乱码），会影响可维护性。
- 右键菜单仍依赖 `prompt` 交互，易造成新手误操作和路径不可发现。
- 本地密钥仍可见于浏览器本地存储（符合当前设计，但需持续安全提示）。

### 3) 验证风险
- 多链路高频变更后，仍需持续执行手工回归（尤其取消生成、草稿隔离、路径定位）。
