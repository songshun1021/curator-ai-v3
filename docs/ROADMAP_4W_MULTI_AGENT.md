# Curator AI 4 周多 Agent 推进路线图（代码基线优先）

## 角色分工

- Agent A（基线与文档）：基线矩阵、文档修订、合并门禁规则。
- Agent B（岗位与简历链路）：JD/定制简历/匹配分析/BOSS/邮件链路稳定。
- Agent C（面试链路）：准备包与复盘闭环、取消与草稿隔离、误判修复。
- Agent D（前端体验）：渲染优先、工具条可达性、新手引导和反馈文案。
- Agent E（QA与发布）：回归矩阵执行、周验收报告、已知问题清单。
- Agent F（安全与开源）：密钥扫描、发布清单、README 新手流程与风险说明。

## 周计划

### Week 1：基线冻结周
- 产出：对齐矩阵、文档修订、P0 清零清单。
- 验收：`pnpm build` 通过；主链路冒烟通过；文档冲突项归零。

### Week 2：机会获取周
- 重点：简历 -> 岗位 -> 文书链路稳定与可见性。
- 验收：新手 10 分钟走通“新建岗位 -> 录入JD -> 生成3类文书”。

### Week 3：通过率提升周
- 重点：准备包与复盘闭环质量，取消生成与草稿策略稳定。
- 验收：复盘误报为 0；准备包可引用历史弱项。

### Week 4：发布准备周
- 重点：开源可交付与新手启动体验。
- 验收：新机器 15 分钟内进入主界面并完成最小链路。

## 统一门禁

1. 每周固定节奏：周一对齐、周三中检、周五验收。
2. 合并门禁：`pnpm build` 必过 + 主链路冒烟必过 + 不新增 P0。
3. 会话门禁：每周更新 `SESSION_COMPACT.md`（完成/验证/待办/建议）。
