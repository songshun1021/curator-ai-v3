# Curator AI 回归矩阵（P0/P1/P2）

## P0（每次合入必测）

1. 模型配置：供应商联动、模型下拉/手输、验证连接、保存刷新回显。
2. 岗位创建：`+新建岗位` 表单创建 `公司-职位-MMDD`，可打开并编辑 `jd.md`。
3. 文书生成：匹配分析/BOSS/邮件可生成，结果路径可定位、可打开、可见。
4. 定制简历：返回非法 JSON 时不落正式文件，给出明确错误提示。
5. 复盘主链路：新建复盘 -> 填写原文 -> 生成报告，无“先填写面试原文”误报。
6. 取消生成：生成中可取消；草稿落盘；正式文件不被覆盖。
7. 构建：`pnpm build` 通过。

## P1（每周至少一次）

1. 渲染优先：`.md`/`.json` 默认渲染，编辑切换与模式记忆正常。
2. 文件树定位：打开路径后自动展开父级、滚动到节点并高亮。
3. 自动保存：JD 与简历编辑刷新不丢失，状态提示正确。
4. 新手引导：首次弹窗、关闭后不重复、AI配置页可手动重开。
5. 多轮生成：连续任务不串线，不污染 notice 与当前文件状态。

## P2（发布前集中验证）

1. 中英混排与长文本阅读体验（Markdown/PDF）。
2. 窄屏下工具条两行可达性。
3. 启动脚本在干净 Windows 环境可运行。
4. 文档一致性抽检（产品规划/手册/AGENTS/SESSION）。
